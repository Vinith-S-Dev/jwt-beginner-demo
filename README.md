A world-class, production-ready implementation of **FastAPI** with **JSON Web Tokens (JWT) Authentication** and **Role-Based Access Control (RBAC)**, paired with a gorgeous, high-fidelity **Glassmorphic Frontend Dashboard** with visual JWT inspector.

---

## ✨ Key Features

- 🔐 **OAuth2 Password Grant Flow**: Seamless client authentication via FastAPI's secure `OAuth2PasswordBearer` scheme.
- 🔑 **CryptContext Passwords**: Robust password hashing and verification utilizing industrial-standard `bcrypt` algorithm.
- 🎫 **JWT Token Lifecycle**: Secured token generation (`HS256`), configurable expiration timer (`30 minutes`), and validation.
- 👥 **Role-Based Access Control (RBAC)**: Fine-grained middleware authorization for `admin` and `employee` roles.
- 🎨 **Ambient Glassmorphism UI**: A stunning, responsive HTML5/Vanilla CSS frontend featuring glowing blur effects, hover animations, and toast systems.
- 🛠️ **Interactive API Console**: Trigger endpoints (`/api/dashboard` and `/api/admin`) directly in-browser to observe HTTP responses and authorization headers.
- 🔬 **Visual JWT Decoder**: Real-time client-side JWT claims viewer that breaks down header and payload segments on screen.
- 🐳 **Docker-Ready**: Instant containerization with single-command `docker-compose` setup.

## 🛠️ Quick Start Guide

### Option A: Local Execution (Fast & Interactive)

#### 1. Setup Virtual Environment
```bash
# Clone the repository (or navigate to directory)
cd Project-folder

# Create a virtual environment
python -m venv venv

# Activate it (Windows)
.\venv\Scripts\activate

# Activate it (macOS/Linux)
source venv/bin/activate
```

#### 2. Install Dependencies
```bash
pip install -r requirements.txt
```

#### 3. Run Development Server
```bash
uvicorn main:app --reload
```
Once launched, open your browser and navigate to **[http://localhost:8000](http://localhost:8000)**.

---

### Option B: Docker Containerized Setup

Build and spin up the complete application container instantly:
```bash
docker compose up --build
```
Access the application on port `8000`: **`http://localhost:8000`**.

## 🔌 API Documentation & Spec

Once running, interactive documentation is auto-generated by OpenAPI at these endpoints:
* **Swagger UI Docs**: `http://localhost:8000/docs`
* **Redoc Spec**: `http://localhost:8000/redoc`

### Endpoints Overview

#### 1. Obtain Auth Token
* **Route**: `POST /api/login`
* **Body Type**: `application/x-www-form-urlencoded`
* **Parameters**: `username`, `password`
* **Response**:
  ```json
  {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5...",
    "token_type": "bearer"
  }
  ```

#### 2. Protected Dashboard
* **Route**: `GET /api/dashboard`
* **Headers**: `Authorization: Bearer <token>`
* **Access**: All authenticated accounts

#### 3. Restricted Admin Command Console
* **Route**: `GET /api/admin`
* **Headers**: `Authorization: Bearer <token>`
* **Access**: Accounts with `admin` role parameter

## 🔒 Security Best Practices Implemented

1. **Environment Variables**: Secret key loads dynamically from environmental config to block source code key leakages.
2. **CORS Middleware Rules**: Structured configuration allowing secure origins.
3. **UTC Expiration Time stamps**: Standardized token validation timestamp.
4. **Clean Errors Handling**: Clear standard error responses prevent system detail disclosures.
